# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Development server (requires Node OpenSSL workaround for Node 16+)
npm run dev      # runs on http://localhost:8080

# Production build
npm run build    # outputs to src/.vuepress/dist/
```

Both scripts already include `set NODE_OPTIONS=--openssl-legacy-provider` to work with Node 22, which is required because VuePress 1.x uses an old webpack that is incompatible with newer OpenSSL.

## Architecture

This is a **VuePress 1.x** documentation site for the Crono BI suite (Analysis, Metadata, ETL, SQL).

### Content structure

- `src/` — VuePress source root
  - `src/analysis/` — Crono Analysis user manual (markdown pages)
  - `src/metadata/` — Crono Metadata user manual
  - `src/etl/` — Crono ETL user manual
  - `src/sql/` — Crono SQL reference (functions, views, language, reference)
    - `src/sql/functions/` — one `.md` file per SQL function (hundreds of files)
    - `src/sql/views/` — one `.md` file per system view
- `src/.vuepress/config.js` — site title, nav links, plugins; imports sidebar from `sidebar.json`
- `src/.vuepress/public/sidebar.json` — **manually maintained** sidebar navigation tree. Must be kept in sync with actual markdown files. When a page is deleted from `src/`, its entry must also be removed from this file or VuePress will log repeated warnings and may hang during build.
- `src/.vuepress/enhanceApp.js` — registers Bootstrap-Vue globally
- `src/.vuepress/components/SectionIndex.vue` — Vue component that fetches `sidebar.json` at runtime and renders child links for the current section

### Auto-generated pages

Many pages under `src/sql/functions/` and `src/sql/views/` have frontmatter `Autogenerated: true`. These are generated by a `cronobot` tool (see `.crono/publish.cmd`) that reads from a SQL Server database (`cronosql.metadata`) and calls stored procedures to regenerate content before building.

### Publish pipeline (`.crono/publish.cmd`)

The full publish flow is managed by `cronobot`:
1. Load GitHub issues into the database
2. Run stored procedures to regenerate function/view pages
3. Update VuePress content (`cronobot sites updatevuepress`)
4. `vuepress build src`
5. `firebase deploy`

### Key constraint

`src/.vuepress/public/sidebar.json` has a UTF-8 BOM and uses `\uXXXX` escapes for non-ASCII characters. Edit it carefully — standard text tools may corrupt the encoding. Use Node.js (`fs.readFileSync`/`writeFileSync`) or PowerShell `[System.IO.File]::ReadAllText` to manipulate it safely.
